# 아조씨 렌탈 랜딩 페이지 스토리텔링 가이드

## 📌 개요

본 문서는 **창업자 중심의 스토리텔링**을 통해 아조씨 렌탈의 가치와 철학을 전달하는 랜딩 페이지 전략을 설명합니다.

---

## 🎬 1. 스토리텔링 구조 (11 Sections)

### 섹션별 목표와 감정 곡선

```
감정 곡선:
         ↗️ 이해 (Sec 0)
        /  ↗️ 공감 (Section 1)
       /  /  ↗️ 감동 (Section 2)
      /  /  /  ↘️ 호기심 (Section 3)
     ↗️  /  /     ↘️ 신뢰 (Section 4)

```

## 🎯 2. 상세 섹션 분석

### Section 0️⃣: 서비스 정의 (슬로건 - 30초)

**목표**: 즉시 이해, 서비스의 본질 전달

**핵심 메시지**:
```
"아조씨를 렌탈합니다.

상담 · 조언 · 동반 · 공감

당신의 이야기를 들어줄
믿을 수 있는 누군가"
```

**설계 요소**:

| 요소 | 설명 | 효과 |
|------|------|------|
| **배경 영상** | 서울 도시 야경 (밤) | 고독감과 희망의 대비 |
| **음성/텍스트** | 명확한 선언 (자신감 있음) | 신뢰, 명확한 가치 제안 |
| **패럴렉스 속도** | 배경 0.1 (거의 정지), 텍스트 0.9 (빠름) | 텍스트 강조 |
| **감정** | 희망, 안정감, 신뢰 | "이 서비스는 나를 위한 거다" |
| **체류 시간** | 약 3-4초 (빠른 인식) | 명확한 메시지 전달 |

**스토리 구조**:
1. **선언**: "아조씨를 렌탈합니다" (서비스 명확화)
2. **가치**: "상담 · 조언 · 동반 · 공감" (무엇을 제공하는가)
3. **약속**: "당신의 이야기를 들어줄 믿을 수 있는 누군가" (고객의 감정적 필요)

**심리적 효과**:
- 🎯 **명확성**: 3초 안에 "뭐하는 서비스"인지 알 수 있음
- 🎯 **신뢰도**: 간결하고 직설적인 메시지 = 성실함
- 🎯 **감정 연결**: "당신의 이야기를 들어줄" = 개인화된 관심

### Section 1️⃣: 창업자의 경험담 (이야기의 시작)

**목표**: 공감 형성, 감정적 연결

**핵심 메시지**:
```
"저는 유튜브를 봤습니다.
최근 몇 년, 댓글 창은 가득했습니다.
중년 남성을 조롱하는 영상들...
'영포티' '아재들' '구닥다리'

그 말들을 읽으면서 생각했습니다.

이게 정말 우리 사회의 모습일까?
중년 남성도 누군가의 아버지고
누군가를 돌보는 존재인데...

나는 그것을 바꾸고 싶었습니다."
```

**설계 요소**:

| 요소 | 설명 | 효과 |
|------|------|------|
| **배경 영상** | 서울 야경, 도시 장면 (밤) | 현대성, 온라인 문화의 어둠 |
| **음성/텍스트** | 창업자의 1인칭 성찰 | 신뢰, 진정성, 문제 의식 |
| **패럴렉스 속도** | 배경 0.3 (느림), 텍스트 0.7 (빠름) | Depth effect |
| **감정** | 분노 → 공감 → 결심의 감정 호 | "이 문제를 공감해" |
| **체류 시간** | 약 5-7초 (자동 스크롤 또는 수동) | 충분한 읽기 및 감정 처리 시간 |

**스토리 구조**:
1. **상황 설정**: 유튜브 댓글 문화와 온라인 조롱
2. **문제 인식**: 중년 남성에 대한 사회적 편견과 조롱
3. **공감**: 중년 남성의 실제 역할과 가치 재인식
4. **결심**: 이런 편견을 바꾸겠다는 사명감

**고객 반응 예상**:
- ✅ 여성 고객: "그런 문제가 있었네, 생각 좋은 사람이네"
- ✅ 중년층: "내 세대를 이해해주는 사람이 있다"
- ✅ 젊은층: "사회적 편견에 대해 생각해보게 돼"

### Section 2️⃣: 왜 만들었는가 (철학 전달 & 창업자 소개)

**목표**: 감동, 신뢰도 구축

**핵심 메시지**:
```
"혼자라고 느끼는 누군가의 말을 들어주는 것.

나이 많은 아조씨의 조언이
누군가의 인생을 바꿀 수 있다는 것.

저는 그것을 믿습니다.
그리고 그것을 실현하고 싶었습니다."
```

**설계 요소**:

| 요소 | 설명 | 효과 |
|------|------|------|
| **창업자 사진** | 따뜻한 표정, 웃음 | 신뢰, 접근성 |
| **텍스트 배치** | 우측에 배치 (이미지 좌측) | 시각적 균형 |
| **패럴렉스 속도** | 사진 0.4, 텍스트 0.8 | 텍스트 강조 |
| **배경색** | 부드러운 흰색/베이지 | 따뜻함 |
| **폰트** | 큰 타이포그래피 (36px 이상) | 가독성, 강조 |

**스토리 구조**:
1. **문제 정의**: "혼자라고 느끼는 누군가" (공감)
2. **해결책**: "아조씨의 조언" (구체성)
3. **영향**: "인생을 바꿀 수 있다" (가치)
4. **약속**: "그것을 실현하겠다" (commitment)

**심리적 효과**:
- 🎯 **신뢰도**: 창업자의 얼굴 노출 = 책임감
- 🎯 **진정성**: 직접적인 말 = 사업이 아닌 mission
- 🎯 **공감**: "누군가"라는 표현 = 포함성

### Section 3️⃣: 서비스 가치 재강조 (Hero Section)

**목표**: 명확한 가치 전달, 행동 유도

**핵심 메시지**:
```
"당신의 이야기를 들어줄
믿을 수 있는 아조씨를
만나보세요"
```

**설계 요소**:

| 요소 | 설명 |
|------|------|
| **CTA 버튼 1** | [아조씨 찾아보기] (Blue) - 고객용 |
| **CTA 버튼 2** | [아조씨 되어보기] (Green) - 공급자용 |
| **배경** | 부드러운 그래디언트 또는 이미지 |
| **패럴렉스** | 배경 0.2 (매우 느림), 텍스트 0.9 (빠름) |

**CTA 전략**:
- **이중 CTA**: 고객/공급자 모두에게 열려있음
- **명확한 레이블**: "찾아보기" vs "되어보기" (구체적 행동)
- **색상 대조**: 파랑 vs 초록 (선택의 명확성)

### Section 4️⃣: 신뢰 지표 (숫자 기반 신뢰)

**목표**: 신뢰도 수치화, 안심

**핵심 메시지**:
```
100% 신원확인  |  1,000+ 예약 완료  |  4.8점 평점
```

**설계 요소**:

| 지표 | 의미 | 메시지 |
|------|------|--------|
| **100% 신원확인** | 모든 아조씨 검증 | "안전합니다" |
| **1,000+ 예약** | 입증된 실적 | "이미 많은 사람이 이용했어요" |
| **4.8점 평점** | 높은 만족도 | "만족한 사람이 많습니다" |

**패럴렉스 효과**:
- 아이콘: 0.5 (중간 속도)
- 숫자: 0.8 (빠른 속도)
- 숫자가 아이콘보다 빨리 움직여서 강조됨

**심리적 효과**:
- 🔢 **신뢰**: 구체적 숫자 = 신뢰
- 🔢 **사회적 증거**: "이미 1,000명이" = 안전신호
- 🔢 **증거**: 평점 = 고객 만족도의 증거

- ✅ **명확성**: 각 단계별 목표 명시
- ✅ **시각성**: 사진으로 직관적 이해

### Section 9️⃣: FAQ (의심 해소)

**목표**: 남은 의문 해결, 마지막 장벽 제거

**주요 질문**:

```
Q1: 정말 안전한가요?
"네, 모든 아조씨는 신원 인증 및
 배경조회를 거쳐 엄격하게 검증됩니다.
 불편한 상황이 있으면 즉시 신고 가능합니다."

Q2: 환불은 어떻게 되나요?
"서비스 이용 후 만족하지 않으시면
 24시간 내에 100% 환불해드립니다."

Q3: 개인정보는 보호되나요?
"고객 정보는 엄격하게 보호됩니다.
 아조씨도 고객을 알 수 없으며,
 이용 후에도 정보는 삭제됩니다."

Q4: 아조씨가 되려면?
"신원 인증 후 3가지 단계를 거칩니다:
 1) 서류 검토
 2) 영상 면접
 3) 안전 교육
 약 1주일이 걸립니다."
```

**설계 요소**:
- Accordion 형식 (펼치기/닫기)
- 첫 번째는 자동 펼친 상태
- 명확한 Q&A 구분
- 길이는 2~4줄 (간결성)

**목표**: 명확한 행동 결정, 가입 유도

```

[지금 시작하기 - 고객]
[아조씨 되기 - 공급자]

────────────────────────────────

SNS로 팔로우하기
[Instagram] [TikTok] [YouTube]
```

**설계 요소**:

| 요소 | 설명 | 효과 |
|------|------|------|
| **최종 메시지** | 감정적, 희망적 톤 | 마음의 준비 |
| **이중 CTA** | 고객 / 공급자 분리 | 명확한 선택 |
| **SNS 링크** | 커뮤니티 강조 | 지속적 관여 |
| **버튼 크기** | 약 200px × 60px | 클릭 용이 |
| **배경** | 최소 contrast (white/light) | 버튼 강조 |

**패럴렉스 효과**:
- 배경: 0.1 (거의 정지)
- 버튼: 0.9 (빠르게)

## 🎨 3. 패럴렉스 스크롤 구현 가이드

### 패럴렉스 속도 조정

**기본 원리**:
```
transform: translateY(scrollPosition * parallaxSpeed)

parallaxSpeed:
- 0.1~0.3: 매우 느림 (배경/보조 요소)
- 0.4~0.6: 중간 (중요 이미지)
- 0.7~0.9: 빠름 (주요 텍스트)
```

**각 섹션별 추천 설정**:

| 섹션 | 배경 | 이미지 | 텍스트 |
|------|------|--------|--------|
| 0: 서비스 정의 | 0.1 | - | 0.9 |
| 1: 창업자 경험 | 0.3 | - | 0.7 |
| 2: 철학 | - | 0.4 | 0.8 |
| 3: 가치 재강조 | 0.2 | - | 0.9 |
| 4: 신뢰 지표 | - | 0.5 | 0.8 |
| 5: 이용 방법 | - | 0.3 | 0.7 |
| 6: 아조씨 프로필 | - | 0.4 | 0.7 |
| 7: 고객 후기 | - | 0.3 | 0.8 |
| 8: 안전 & 신뢰 | 0.2 | - | 0.8 |
| 9: FAQ | - | - | 0.6 |
| 10: 최종 CTA | 0.1 | - | 0.9 |

### React 구현 예시

```typescript
import { useState, useEffect } from 'react';

interface ParallexSectionProps {
  backgroundSpeed: number;
  foregroundSpeed: number;
  children: React.ReactNode;
}

export function ParallaxSection({
  backgroundSpeed,
  foregroundSpeed,
  children
}: ParallexSectionProps) {
  const [scrollY, setScrollY] = useState(0);

  useEffect(() => {
    const handleScroll = () => setScrollY(window.scrollY);
    window.addEventListener('scroll', handleScroll);
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);

  return (
    <div className="relative h-screen overflow-hidden">
      {/* Background */}
      <div
        className="absolute inset-0"
        style={{
          transform: `translateY(${scrollY * backgroundSpeed}px)`,
          backgroundImage: 'url(...)',
          backgroundSize: 'cover',
        }}
      />

      {/* Foreground Content */}
      <div
        className="absolute inset-0 flex items-center justify-center"
        style={{
          transform: `translateY(${scrollY * foregroundSpeed}px)`,
        }}
      >
        {children}
      </div>
    </div>
  );
}
```

## 📱 4. 모바일 최적화 전략

### 모바일에서의 패럴렉스

**문제점**:
- 모바일에서 복잡한 패럴렉스는 성능 저하
- 버벅임, 배터리 소모 증가
- 좋은 UX 아님

**해결책**:
1. **속도 감소**: 0.2배 이하로 조정
2. **선택적 활성화**: 화면 크기 768px 이상에서만 활성화
3. **대체 효과**: 간단한 fade-in/fade-out 효과

```typescript
const isDesktop = window.innerWidth >= 768;
const parallaxSpeed = isDesktop ? 0.7 : 0.1;
```

### 모바일 터치 최적화

| 요소 | 모바일 조정 |
|------|----------|
| **버튼 크기** | 48px × 48px 최소 |
| **텍스트 크기** | 본문 16px 이상 |
| **간격** | 더 넉넉하게 (padding/margin 증가) |
| **한 줄 너비** | 40~50자 (가독성) |
