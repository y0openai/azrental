# 렌탈 아재 웹사이트 기획서

## 📌 프로젝트 개요

**프로젝트명**: 렌탈 아재 (가칭)
**비즈니스 모델**: 양방향 마켓플레이스 (Two-Sided Marketplace)
**타겟 시장**: 대한민국

**핵심 가치 제안**:
- **고객**: 신뢰할 수 있는 중년 남성(아재)과 의미 있는 시간을 통해 정서적 안정감과 실질적 조언을 얻음
- **아재 공급자**: 새로운 역할 창출과 추가 소득 기회를 통해 사회 참여 확대
- **사회**: 고립된 1인 가구와 고령화 시대의 정서적 사회 문제 해결

## 🛠️ 기술 스펙 (확정)

### Backend (백엔드)
- **Platform**: Google Firebase (Firestore, Authentication, Cloud Functions)
- **Real-time**: Google Realtime Database (채팅 시스템)
- **Notifications**: Google FCM (Firebase Cloud Messaging)

### Frontend (프론트엔드)
- **Framework**: Next.js 14+ (App Router)
- **UI**: Tailwind CSS + shadcn/ui
- **State Management**: Zustand + React Query
- **Language**: TypeScript

### Unique Features (특색 있는 기능)
- **랜딩 페이지**: 패럴렉스 스크롤 기술 활용
  - 동영상 (서비스 소개 영상)
  - 사진 (고객 후기, 아재 프로필)
  - 텍스트 (설명, CTA)
  - 믹스 형식으로 세련되게 서비스 소개

### Hosting & Deployment
- **Frontend**: Vercel (Next.js 최적화)
- **Backend**: Firebase Cloud (자동 관리)
- **Database**: Firestore (NoSQL, Real-time)
- **Storage**: Firebase Storage (사진, 문서)

## 🎯 브랜드 네이밍 & 포지셔닝

### 최종 브랜드명 확정

**선택**: **아조씨 렌탈** ✅

**선택 근거**:
- "아조씨": 최근 한국에서 젊고 느낌있는 아저씨를 부르는 트렌드 단어
- "친근하고 캐주얼": "렌탈 아재"보다 더 현대적
- "세대 공감": 젊은층과 중년층 모두에게 긍정적 이미지
- "마케팅 효과": 기존 "아저씨 렌탈" 대비 신선함

### 이전 후보안

| 브랜드명 | 특징 | 타겟 감정 | 평가 |
|---------|------|---------|------|
| **렌탈 아재** | 원본 모델 직역, 명확함 | 접근성, 친근감 | 다소 딱딱함 |
| **아재매칭** | 마칭 개념 강조 | 신뢰, 연결 | 마칭 연상 문제 |
| **어른의 시간** | 감정적, 성숙함 | 따뜻함, 신뢰도 | 포괄적 과묵함 |
| **아저씨 네트워크** | 커뮤니티 강조 | 안전성, 신뢰성 | 딱딱하고 형식적 |
| **아조씨 렌탈** | 트렌디, 친근함 | 세대공감, 신뢰 | ✅ **선택** |

### 브랜드 톤 & 매너

**톤**: 따뜻하고 신뢰할 수 있는, 다정함
- 과도하게 캐주얼하지 않음
- 전문성과 친근감의 균형
- 안전과 신뢰 강조

**비주얼 톤**:
- 색상: 따뜻한 파랑(신뢰, 안정) + 밝은 그린(안전, 긍정)
- 폰트: Pretendard (모던함과 한글 최적화)
- 이미지: 실제 사람들의 따뜻한 상호작용

## 📊 2. 메뉴 구조 (Information Architecture)

### 2.1 Top-Level Navigation (모든 페이지 공통)

**Header Navigation**:
```
┌─────────────────────────────────────────────────────┐
│ [로고] [검색 바] [위시리스트] [메시지] [프로필▼] │
└─────────────────────────────────────────────────────┘
```

**Desktop (주 네비게이션)**:
```
/ | 아재 찾기 | 서비스 소개 | 블로그 | 고객지원 | [로그인]
```

**Mobile (하단 탭 네비게이션 - 한국 UX 표준)**:
```
┌─────────────────────────────────────┐
│ 홈    │ 탐색  │ 예약  │ 메시지 │ 내정보 │
│  🏠   │  🔍   │  📅   │   💬   │  👤  │
└─────────────────────────────────────┘
```

### 2.2 사용자별 메뉴 구조

**비로그인 사용자**:
- 홈 → 아재 탐색 → 상세 (로그인 필요 메시지) → 로그인 → 예약

**고객**:
- 탭 1: 홈 (추천, 최근 본 아재)
- 탭 2: 아재 탐색 (검색, 필터)
- 탭 3: 내 예약
- 탭 4: 메시지
- 탭 5: 내 정보 (프로필, 설정, 수시 정보)

**아재**:
- 탭 1: 대시보드 (수익, 일정, 평점)
- 탭 2: 예약 요청
- 탭 3: 일정 관리
- 탭 4: 메시지
- 탭 5: 수익/정산

## 📱 4. 주요 사용자 여정 (User Journey)

### 4.1 고객 사용자 여정: 아재 예약하기

```
1️⃣ 인식 (Awareness)
   ├─ 검색: "혼자 있어서 힘들어" → "렌탈 아재" 검색
   ├─ 유입: Google, Naver, Kakao, Instagram 광고
   └─ 랜딩: 홈페이지 도착

2️⃣ 탐색 (Exploration)
   ├─ 홈페이지에서 "이용 방법" 영상 시청
   ├─ FAQ 읽기
   ├─ 고객 후기 (testimonial) 확인
   └─ 신뢰도 평가 (안전 정책, 신원 인증 배지)

3️⃣ 관심 (Interest)
   ├─ "아재 찾아보기" 클릭
   ├─ 아재 목록 탐색
   ├─ 지역별 필터 (강남구 아재들)
   ├─ 서비스 필터 (상담, 취미 동반)
   └─ 평점 확인 (4.8점 이상)

4️⃣ 선택 (Selection)
   ├─ 마음에 드는 아재 프로필 클릭
   ├─ 상세 정보 확인
   │  ├─ 프로필 사진 갤러리
   │  ├─ 자기소개 읽기
   │  ├─ 받은 리뷰 5개 확인
   │  └─ 가용 시간 캘린더 확인
   ├─ 즐겨찾기 (찜하기)
   └─ "예약하기" 버튼 클릭 (로그인 필요 시 로그인)

5️⃣ 예약 (Booking)
   Step 1️⃣: 날짜/시간 선택
   ├─ 캘린더에서 원하는 날짜 선택
   ├─ 가용한 시간대 표시 (초록색)
   ├─ 오후 2:00 선택
   └─ "다음" 버튼

   Step 2️⃣: 서비스 상세
   ├─ 서비스 종류 선택 (상담, 취미 동반 등)
   ├─ 만날 장소 선택 (카페, 공원, 온라인, 고객 위치)
   ├─ 장소가 고객 위치이면 주소 입력
   ├─ 시간 선택 (1~4시간, 1시간 단위)
   └─ "다음" 버튼

   Step 3️⃣: 요청 정보
   ├─ 기대사항/요청사항 입력 (선택)
   │  └─ "취업 준비 중이라서 조언 부탁합니다"
   ├─ 익명 여부 선택 (기본값: 익명)
   └─ "다음" 버튼

   Step 4️⃣: 결제
   ├─ 가격 정보 표시
   │  ├─ 시간당 요금: 20,000원
   │  ├─ × 시간: 2시간 = 40,000원
   │  ├─ 플랫폼 수수료: 4,000원 (10%)
   │  └─ **총액: 44,000원**
   ├─ 결제 수단 선택 (Kakao Pay 추천)
   ├─ 이용약관 & 개인정보 동의 체크
   └─ "[44,000원 결제하기]" 버튼

   Step 5️⃣: 확정
   ├─ 결제 성공 화면
   ├─ 예약 확정 메시지
   ├─ 예약 번호 및 아재 정보 표시
   ├─ "아재에게 메시지 보내기" 버튼 (선택)
   └─ "내 예약으로 돌아가기" 버튼

6️⃣ 사전 소통 (Pre-Meeting)
   ├─ 예약 후 메시지 박스 활성화
   ├─ 아재 프로필 이미지, 이름, 연락처 표시
   ├─ 예약자 메시지: "안녕하세요! 기대됩니다"
   ├─ 아재 메시지: "안녕하세요! [카페명]에서 만나요"
   ├─ 만나는 장소 최종 확정
   └─ 1일 전 알림: "내일 오후 2시 만남이 예정되어 있습니다"

7️⃣ 서비스 당일 (Service Day)
   ├─ 1시간 전 알림: "1시간 후 만남이 시작됩니다"
   ├─ 만나기
   ├─ 서비스 진행 (상담 2시간)
   └─ 완료

8️⃣ 사후 처리 (Post-Service)
   ├─ 아재가 "서비스 완료" 표시
   ├─ 고객이 "서비스 확인" 표시 (24시간 내)
   ├─ Escrow → 아재 정산 진행
   ├─ 리뷰 작성 알림 (이메일/푸시)
   └─ 리뷰 작성 화면으로 유도

9️⃣ 평가 (Review)
   ├─ 별점 5개 중 5개 선택
   ├─ 리뷰 작성: "정말 좋은 조언 감사합니다! 면접 준비가 훨씬 확신이 서네요."
   ├─ 태그 선택: #친절함 #경청 #시간엄수
   ├─ 사진 첨부 (선택)
   └─ "리뷰 완료" 버튼

🔟 재방문 (Return)
   ├─ 홈 화면에 "김철수님과의 만남이 좋았나요? 재예약하기"
   ├─ 아재 즐겨찾기에 추가됨
   ├─ 아재 목록에서 "★ 찜 해제" 표시
   └─ 다시 만날 준비 완료
```

**고객 감정 곡선**:
- 처음: 의심적, 안전 걱정
- 아재 선택 후: 기대감, 설레임
- 만날 준비: 안정감, 신뢰
- 만남 중: 긍정적, 편안함
- 후회: 감사함, 재방문 의욕

### 4.2 아재 사용자 여정: 가입 및 예약 관리

```
1️⃣ 인식 (Awareness)
   ├─ "추가 소득 생각 중이야" → 검색
   └─ 렌탈 아재 사이트 발견

2️⃣ 고려 (Consideration)
   ├─ 홈페이지에서 "아재 지원하기" 클릭
   ├─ 아재 지원 가이드 읽기
   ├─ FAQ 확인 (나도 할 수 있을까?)
   └─ "지원하기" 버튼 클릭

3️⃣ 신청 (Application)
   ├─ 아재 지원서 작성
   │  ├─ 기본 정보 입력
   │  │  ├─ 이름, 나이, 위치
   │  │  ├─ 직업, 경력
   │  │  └─ 기본 사진
   │  ├─ 신원 인증
   │  │  ├─ 신분증 업로드
   │  │  ├─ 셀카 (신분증과 함께)
   │  │  └─ 배경조회 동의
   │  ├─ 프로필 작성
   │  │  ├─ 자기소개 (500자)
   │  │  ├─ 스킬 태그 (상담, 면접 연습, 취미 동반 등)
   │  │  └─ 사진 업로드 (최대 5장)
   │  └─ 가용성 & 요금
   │     ├─ 주간 스케줄 설정
   │     ├─ 시간당 요금 설정 (기본값 제안: 20,000원)
   │     └─ 제출
   │
   └─ 지원서 제출 완료

4️⃣ 검증 (Verification)
   ├─ 관리자 검토 (48-72시간)
   │  ├─ 신분증 확인
   │  ├─ 배경조회 진행
   │  └─ 요청 시 영상 면접
   ├─ 이메일 상태 업데이트
   └─ "검증 완료" 또는 "추가 정보 필요" 연락

5️⃣ 승인 (Approval)
   ├─ 승인 이메일 수신
   ├─ 대시보드 접근 권한 활성화
   ├─ 온보딩 비디오 시청 (안전 가이드)
   ├─ 안전 가이드 퀴즈 완료
   └─ 프로필 공개 (고객 검색 결과에 표시)

6️⃣ 첫 예약 (First Booking)
   ├─ 대시보드에서 첫 예약 요청 수신
   │  ├─ 고객명: "익명_1234"
   │  ├─ 날짜: 2026년 1월 15일 (수) 오후 2시
   │  ├─ 시간: 2시간
   │  ├─ 서비스: 상담 (취업 준비)
   │  ├─ 장소: 강남역 스타벅스
   │  └─ 요청사항: "취업 준비 중입니다. 조언 부탁해요"
   │
   ├─ "[수락] [거절]" 버튼 선택
   │
   └─ 수락 클릭 후
      ├─ 고객에게 수락 알림 전송
      ├─ 메시지 박스 활성화
      ├─ 메시지: "안녕하세요! 기대합니다 😊"
      ├─ 장소 최종 확인
      └─ 일정에 예약 추가

7️⃣ 서비스 당일 (Service Day)
   ├─ 1시간 전 알림: "1시간 후 만남이 시작됩니다"
   ├─ 만나기
   ├─ 서비스 진행 (상담 2시간)
   └─ 완료

8️⃣ 서비스 완료 (Completion)
   ├─ 아재가 "서비스 완료" 표시
   │  └─ 버튼: "[서비스 완료]" 클릭
   │
   ├─ 시스템이 정산 대기 상태로 전환
   │  └─ 고객 확인 대기 (24시간)
```

## 🇰🇷 6. 한국 시장 UX 특성

### 6.1 선호하는 디자인 패턴

| 요소 | 한국 특성 | 구현 방식 |
|------|---------|---------|
| **하단 네비게이션** | 모바일 표준 패턴 | 탭 5개: 홈/탐색/예약/메시지/프로필 |
| **결제 방식** | 모바일 결제 선호 | Kakao Pay 우선, Toss 2순, Naver 3순 |
| **폰트** | 한글 최적화 필수 | Pretendard (15px 본문) |
| **색상** | 신뢰+안전 강조 | 파랑(신뢰) + 초록(안전) |
| **이용약관** | 명확성 강조 | 풍선 도움말, 약관 내 하이라이트 |
| **주소 입력** | API 자동완성 | Kakao or Naver Address API |

### 6.2 문화적 고려사항

- **존댓글 사용**: "예약하셨습니다" (정중함 강조)
- **익명성**: 기본값 익명 (개인정보 보호 선호)
- **신뢰 표시**: "신원 확인됨" 배지 필수
- **안전 강조**: 신고, 환불, 취소 정책 명확히
- **긍정적 표현**: "함께 성장해요" (공동 성장 강조)
